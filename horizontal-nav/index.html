How to make a horizontal navigation bar

Navgation is a list of links so its a good idea to make your navigation a list. Wrap your list in a <nav> tag because thats what its for (defines main navagation block)

<nav>
    <ul>
        <li><a href="#">About us</a></li>
        <li><a href="#">Portfolio</a></li>
        <li><a href="#">Our clients</a></li>
        <li><a href="#">Contact us</a></li>
    </ul>
</nav>

So this will look like this:

  • About us
  • Portfolio
  • Our clients
  • Contact us

Lets make it horizontal:

I'm going to add class="horizontal" to the nav tag to isolate this for styling from other lists.

<nav class="horizontal">
    <ul>
        <li><a href="#">About us</a></li>
        <li><a href="#">Portfolio</a></li>
        <li><a href="#">Our clients</a></li>
        <li><a href="#">Contact us</a></li>
    </ul>
</nav>

We have three options for making the list items sit side by side, each option is valid and you might wnat to use each depending on whatyou are trying to achieve style-wise

Option one - display: inline;

.horizontal li {
    display: inline;
}
  • About us
  • Portfolio
  • Our clients
  • Contact us

This is good option, but if you want to give your li's height or top and bottom padding you'll come unstuck since they are now inline you can't.

Option two - display: inline-block;

.horizontal li {
    display: inline-block;
}
  • About us
  • Portfolio
  • Our clients
  • Contact us

Now you can give your li's height, but inline and inline-block elements have little space between them, which if you don't want can be hard to get rid of.

Option three - float: left;

.horizontal li {
    float: left;
}

This option is fine too, but don't forget to clear floats!

Let's use this option.

So you can see that making the <li> inline or inline block seems to rid us of the bullets, I don't think this is really whats happening, its probably just outside the visible box around the <li>.

Using float has made our <li>'s bullets float over the top of the previous <li>. This is because the bullets sit outside the content box.

Lets get rid of those pesky bullets, add some spacing between the links and remove the default <ul> left padding.

.horizontal ul {
    padding: 0;
}
.horizontal li {
    float: left;
    list-style: none;
    margin-right: 20px;
}

Because we are floating our <li>'s our <ul> has collapsed. See how this text is wrapping around it now and setting background color to the <ul> does nothing.

We could use a clearfix on our <ul>, or we could also float our <ul> left and give it a 100% width to fix this.

.horizontal ul {
    float: left;
    width: 100%;
    padding: 0;
}
.horizontal li {
    float: left;

Let's make it look a little nicer and give our links a hover state! Hover on them to see it in action.

I need to switch to the inline-block option because I'm going to want to give the <li>'s some height via padding on the <a> tags.

.horizontal ul {
    background: #178CA6;
    text-align: right;
    padding: 0;
}
.horizontal li {
    display: inline-block;
    list-style: none;
    margin-right: 0; 
}
.horizontal a {
    display: block; /* fill li space */
    cursor: pointer; /* need this to restore pointer which gets lost when display: block; on a tag */
    padding: 1em 0.8em;
    color: #fff;
    text-decoration: none;
}
.horizontal a:hover {
    background: rgba(0,0,0,0.2);
}

Other stuff you might want to do:

Add an evenly spaced line between them.

If your padding left and right of the <a> tag is even then you can add a border right to either your <a> tag or <a> then use a :last-child pseudo selector to remove the last border.

.horizontal ul {
    background: #178CA6;
    text-align: right;
    padding: 0;
}
.horizontal li {
    display: inline-block;
    list-style: none;
    margin-right: 0; 
}
.horizontal a {
    display: block; /* fill li space */
    cursor: pointer; /* need this to restore pointer which gets lost when display: block; on a tag */
    padding: 1em 0.8em;
    color: #fff;
    text-decoration: none;
    border-right: 1px solid #0F5B6C;
}
.horizontal a:hover {
    background: rgba(0,0,0,0.2);
}
.horizontal a:last-child {
    border-right: 0;
}

Can you see the problematic inline/inline-block spacing between the li's, its really obvious when you hover!

One way of getting rid of this is in chrome/safari and firefox is to use something from yahoo's yui grid: set letter-spacing: -0.31em; on the <ul> and letter-spacing: normal; on the <li>. I wonder if it works in ie and elswhere.

.horizontal ul {
    background: #178CA6;
    text-align: right;
    padding: 0;
    letter-spacing: -0.31em;
}
.horizontal li {
    display: inline-block;
    list-style: none;
    margin-right: 0;
    letter-spacing: normal;
}
.horizontal a {
    display: block; /* fill li space */
    cursor: pointer; /* need this to restore pointer which gets lost when display: block; on a tag */
    padding: 1em 0.8em;
    color: #fff;
    text-decoration: none;
    border-right: 1px solid #0F5B6C;
}
.horizontal a:hover {
    background: rgba(0,0,0,0.2);
}
.horizontal a:last-child {
    border-right: 0;
}
