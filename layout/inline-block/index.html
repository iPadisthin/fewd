Inline-block for layout

In-line blocks are great but traditionally haven't been used for layout because of the hard to control white space between them.

    <h2>I'm 25% wide.</h2>
    <p>This is</p>
    <p>Some content</p>
    <p>For block one</p>
</div>
<div class="one-quarter">
    <h2>I'm 25% wide.</h2>

    <p>Some content</p>

    <p>Extra line</p>
</div>

    <p>This is</p>
    <p>Some content three</p>
</div>
<div class="one-quarter">
    <h2>I'm 25% wide.</h2>
    <p>This is content</p>
</div>

The CSS

                
.one-quarter {
    display: inline-block;
    width: 25%;
}

So you can see that even though each of these blocks are 25% wide they don't all fit on the one line.

This is because of that white space that appears between each inline-block that each browser applys but we can't be sure how wide it is in each browser.

The other thing you might notice is that inline-blocks align at there bottom edge by default.

Let's make them align at their top edge.

                
.one-quarter {
    display: inline-block;
    width: 25%;
    vertical-align: top;
}

How do we get rid of that white space!

Here's a technique I like to use because its the most robust I've seen - https://github.com/patrickkunka/zero-width

You put your inline-blocks inside a container, just like we did for our float layout:

                
<div class="layout">
    <div class="one-quarter">
        <h2>I'm 25% wide.</h2>
        <p>This is</p>
        <p>Some content</p>
        <p>For block one</p>
    </div>
    <div class="one-quarter">
        <h2>I'm 25% wide.</h2>
        <p>This is</p>
        <p>Some content</p>
        <p>For block two</p>
        <p>Extra line</p>
    </div>
    <div class="one-quarter">
        <h2>I'm 25% wide.</h2>
        <p>This is</p>
        <p>Some content three</p>
    </div>
    <div class="one-quarter">
        <h2>I'm 25% wide.</h2>
        <p>This is content</p>
    </div>
</div>

You set the container's (<div class="layout">) font-family to a "zero width" font and font-size to zero.

You reset the font-family for the inline-blocks within your container to the default font-family you are using and the font-size to the base font-size you are using.

                
@import url('https://cdn.jsdelivr.net/font-zero-width/latest/zero-width.css');
/* import zero width font - must be at op of CSS page */

.layout {
    font-family: 'zero-width'; /* set font-family to zero width */
    font-size: 0; /* set font-size to 0 */
}
.one-quarter {

    font-family: 'helvetica'; /* set font-family back to your default */
    font-size: 1rem; /* set font-size to 1rem which is 1 * body font-size  */

    display: inline-block;
    width: 25%;
    vertical-align: top;
}
